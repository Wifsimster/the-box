Profile Layout — Personas Meeting Internal

branch: claude/improve-profile-layout-03M3S · date: 2026-05-21 · audience: engineering

Three personas were spawned in parallel to review the profile page after a year of feature additions left it as a 499-line vertical wall of cards. Each had a narrow lens — UX/information-architecture, frontend implementation, and mobile QA — and reported back with prioritised, file-level findings. This note captures the synthesis, the decisions, and what shipped.

Verdict

Split the profile into four URL-persisted tabs. All three personas independently picked tabs over accordions or anchored sections. The page today is one space-y-8 stack of ten Cards with identical visual weight: GDPR consent toggles compete with a 479-line streamer-developer console, achievements (the actual reason most users visit) sit six cards below the fold, and the staggered Framer-Motion delays (0.45, 0.46, 0.47, …) read like an unspoken admission that the ordering already lost.

Starting point

ProfilePage.tsx renders, in order: identity-and-stats header card, Referral, EmailConsent, PushNotification, StreamerKit (479 LOC + 396 webhooks + 128 recipes), GeoContributor, AdvancedStats (premium-only), ThemeSwitcher, Replay-Tour Card, Achievements grid. Each is its own motion.div with hand-tuned delay; each of the four stat tiles re-declares TooltipProvider independently; the loading skeleton bears no resemblance to the real layout. Total scroll height on a 390-pt iPhone is six to seven viewports for a free user.

Personas & Findings

UX / Information Architecture Re-architect

daily-engagement loop · progressive disclosure

The 12-card stack gives identity, GDPR, viral-growth, and a developer console identical billing. StreamerKitCard alone eats roughly 40 % of total scroll for the <2 % of users who use it; for everyone else it pushes achievements off the screen entirely. Premium perks are invisible to free users (no upsell affordance where AdvancedStatsPanel would render).

  • P0 Tabs, URL-anchored — Overview · Account · Creator · Customize. Tabs beat accordions (no double-click), beat anchor-nav (long page still loads everything), beat two-column (StreamerKit needs full width).
  • P0 Identity card & achievements grid live in Overview — that’s why 95 % of users open this page.
  • P1 Group settings: EmailConsentCard, PushNotificationCard, ReferralCard → Account. ThemeSwitcher, AdvancedStatsPanel, Replay Tour → Customize. StreamerKitCard + GeoContributorCard → Creator.
  • P2 Eventually relocate Streamer Kit to its own /streamer route — it’s a dev console, not profile content.

Frontend Engineer Build plan

React 19 · Tailwind v4 · shadcn/ui · react-router 7

Use the existing shadcn tabs primitive (already in components/ui/tabs.tsx). Persist the active tab in a ?tab= search param — back button works, deep-links work, no route-table churn. Lazy-load StreamerKit (479 LOC), GeoContributor, AdvancedStats and ThemeSwitcher per tab so a free-tier overview render no longer ships them. Extract StatTile to kill the four duplicated TooltipProvider blocks; extract ProfileSection to kill the ten motion.div delay-boilerplates.

  • P0 StatTile, ProfileSection, ProfileHeaderCard, ProfileTabs, ProfileSkeleton components.
  • P0 React.lazy on StreamerKit + AdvancedStats + GeoContributor + ThemeSwitcher; Overview eager (LCP path).
  • P1 useSearchParams for tab state with overview default.
  • P1 Skeleton mirrors the real layout (header + stat row + tab strip + 6-card grid) instead of four anonymous blocks.

Mobile QA / Real-Device Scroll & reach

390×844 · bottom-nav clearance · touch tooltips

Six to seven viewports of scroll today. The 2×2 stat grid stacked below the avatar consumes a whole viewport before any actionable content. Stat tiles are cursor-help divs wrapping Tooltip — on touch devices the tooltip body is unreachable. The page has no bottom padding for the persistent BottomNav, so the last achievement card sits flush behind it.

  • P0 Tabs collapse 80 % of the scroll burden — non-Overview content is opt-in.
  • P1 Add pb-[var(--bottom-nav-h)] to the page container so achievements don’t hide behind the bottom bar.
  • P2 Swap stat-tile tooltips for a Popover (tap-to-open) in a follow-up — out of scope here.
  • P2 Consider gating CubeBackground behind md: — pure decoration on a scroll-heavy page.

Decisions

Scope — consensus P0 + P1 set. Four-tab split with URL state, StatTile + ProfileSection + ProfileHeaderCard + ProfileSkeleton extraction, lazy-loaded heavy panels, and bottom-nav padding. P2 follow-ups (tap-tooltips, CubeBackground gating, /streamer relocation) are out of scope.
Tab destinations — Overview · Account · Creator · Customize. Overview is the default landing tab and renders the identity card + achievements grid; everything else is one tap away.
