Personas & Findings

Backend / Queue Orchestration SAFE

BullMQ · maps pipeline

Traced every maps-fetch-*.ts worker. Conclusion: the comment in maps-pipeline-logic.ts:80 claiming “the child re-enqueues the orchestrator on completion (success or failure)” is stale. In reality, runSourceFetch in maps-fetch-runtime.ts:126-135 calls recordAttempt(...) and throw err on the rethrow branch — the post-success advancePipeline() at line 140 is unreachable on throw. The PR’s new failed handler is therefore the only advance path on terminal failure.

BullMQ behavior also checks out: attempts: 5 is configured in queues.ts:109, and BullMQ fires failed with attemptsMade === opts.attempts on terminal failure, so the attemptsMade < maxAttempts guard is off-by-one-free.

Actions:

  • Fix the stale comment in maps-pipeline-logic.ts so the next reader doesn’t chase the same ghost.
  • Extract shouldAdvanceAfterFailure(job) as a pure predicate, so QA can unit-test the off-by-one.
  • Skipped: tightening the dedupe jobId to be job.id-based. Marginal value once the race is disproven.

Frontend / UX / A11y 2 BUGS

ReviewWorkspace · keyboard shortcuts

Bug A — Escape collides with Radix Dialog (ReviewWorkspace.tsx:498). Radix DismissableLayer attaches its Escape listener on ownerDocument and calls preventDefault() when handling Escape. Document bubble-phase listeners fire before window listeners, so Radix closes the Reject/Demote dialog — then our window listener still fires and calls onCloseDetail, closing the workspace underneath the user.

Bug B — Shift+A triggers Promote (ReviewWorkspace.tsx:451). The modifier guard rejects Cmd/Ctrl/Alt but not Shift, and the switch matches both 'a' and 'A'. KeyHints displays <kbd>A</kbd>, which conventionally implies unshifted.

Minor:

  • Switch useLayoutEffect ref-sync to write-during-render (idiomatic since React 18 concurrent renderer).
  • Restore the 'Retour à la liste' default-value second arg on t('admin.geo.nav.backToList') to match prior code and ops-string convention.
  • Sticky ActionBar resolves against the document scroll (no ancestor has overflow or max-h). That’s the intended behavior — no action.

QA / Test Strategy 2 tests

node:test · Playwright

Backend uses node --test via tsx (8 existing *.test.ts). Frontend has zero unit-test infra — only Playwright + axe. RTL is not installed and isn’t worth adding for one hook.

  • Backend: unit-test the extracted shouldAdvanceAfterFailure predicate. Three cases — mid-retry returns false, terminal returns true, wrong kind returns false. Lives at packages/backend/src/infrastructure/queue/workers/geo.worker.test.ts.
