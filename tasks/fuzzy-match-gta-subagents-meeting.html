Fuzzy-Match: GTA review

Date 2026-05-20 · Trigger live game session vs Grand Theft Auto: Vice City · Attendees Fuzzy-Match Specialist · Game-Domain Expert · Architect · QA · Facilitator Claude

Two bugs surfaced in one game. One false positive (grand thief auto 3 awarded +125 for Vice City) and one false negative (gta vice city rejected). Both live in packages/backend/src/domain/services/fuzzy-match.service.ts. The fix is two surgical additions: a new guard in the base-name path, and a generic derived-acronym expansion at the top of step 3. No rewrite, no per-game alias dumping ground.

2. The personas

Four agents ran in parallel. Their unedited findings are below; the synthesis is in section 3.

Fuzzy-Match Specialist Forensic trace

Bug A — "grand thief auto 3" accepted for Vice City

Parsed titles:

input  = { seriesName: "grand thief auto", seriesNumber: 3,    subtitle: null }
target = { seriesName: "Grand Theft Auto", seriesNumber: null, subtitle: "Vice City",
           baseName:   "Grand Theft Auto" }

Control flow lands in the base-name path at lines 528–582. baseNameSimilarity = JW("grand thief auto 3", "grand theft auto") clears the 0.90 bar (long shared prefix, one-letter delta on "thief"/"theft", trailing "3" is small noise). Both existing guards then short-circuit on targetParsed.seriesNumber !== null — which is false here, because Vice City has no number. hasWrongSeriesNumber(3, null) also returns false (one side null). The function returns true.

Bug B — "gta vice city" rejected for Vice City

Parsed titles:

input  = { seriesName: "gta vice city", seriesNumber: null, subtitle: null }
target = { ... subtitle: "Vice City", baseName: "Grand Theft Auto" }

Because "gta vice city".split(' ').length > 2 is true, hasSeriesIndicators is true and the parser assigns the whole input to seriesName — so isSubtitleOnly() returns false and the subtitle-only branch (3a) is skipped. The base-name path fails because JW("gta vice city", "grand theft auto") ≈ 0.45. Token-sort fails. Full-fuzzy JW("gta vice city", "grand theft auto vice city") ≈ 0.78 — below the 0.88 floor. No match.

Architect Minimum-change fix

Fix A — guard the base-name path

In fuzzy-match.service.ts, inside the if (targetParsed.baseName && targetParsed.subtitle) block (around lines 553–575), add a third guard after the existing two: 

// Numbered input vs a subtitled, unnumbered target: the input is naming a
// different entry in the same franchise. Reject unless the input also looks
// like the subtitle.
if (
  targetParsed.seriesNumber === null &&
  targetParsed.subtitle !== null &&
  inputParsed.seriesNumber !== null
) {
  const subtitleSim = jaroWinkler(
    normalizeForFuzzy(input),
    normalizeForFuzzy(targetParsed.subtitle)
  )
  if (subtitleSim < SUBTITLE_THRESHOLD) {
    log.debug({ input, gameName }, 'rejected - numbered input on subtitled-unnumbered target')
    return false
  }
}

Fix B — derived-acronym expansion

Algorithmic, not data. Add a helper near parseGameTitle and call it once at the top of step 3 in isMatchEnhanced: 

function deriveAcronym(name: string): string | null {
  const toks = tokenize(normalizeForFuzzy(name))
                 .filter(t => !STOP_TOKENS.has(t) && t.length >= 1)
  if (toks.length < 2) return null
  const acronym = toks.map(t => t[0]).join('')
  return acronym.length >= 2 ? acronym : null
}

// At the top of step 3, after parsing both sides, guarded by !expandedRetry:
const acronym = deriveAcronym(targetParsed.baseName ?? gameName)
const inputTokens = tokenize(normalizedInput)
if (acronym && inputTokens[0] === acronym) {
  const expanded = [
    normalizeForFuzzy(targetParsed.baseName ?? gameName),
    ...inputTokens.slice(1),
  ].join(' ')
  if (isMatchEnhanced(expanded, gameName, aliases, log, /*expandedRetry=*/true)) return true
}

Stop-token-filtered initials of the target's base name. For "Grand Theft Auto" → "gta"; for "The Legend of Zelda" → "lz" (lowercased — still a clean acronym after stop-word filtering). Single-word names ("Cuphead", "Skyrim") skip the path entirely.

The expandedRetry flag caps recursion at depth 1. The expanded form must still clear the existing thresholds; we're not opening a new low door — we're letting an unrecognised initialism reach the door it already deserves.

Why not just add "gta" as an alias? Because we'd then need to do the same for every multi-word franchise (TLOU, BotW, MGS, FF, CoD, SMB, …). The derived-acronym rule handles them in one place. Irregular shortenings like "PUBG" or "CS:GO" still belong in per-game alias arrays.

What could break (and why it doesn't)

  • witcher 3 → The Witcher 3: Wild Hunt — target has seriesNumber=3, Fix A's first condition is false. Unaffected.
  • paper mario → Paper Mario: The Thousand-Year Door — input has no number, Fix A skips. Unaffected.
  • cuphead → Cuphead: The Delicious Last Course — rejected earlier by isBaseNameOnlyMatch; reaches neither fix. Unaffected.
  • Fix B expansion can only add a passing branch — never bypass an existing reject, because the expanded form still runs the full algorithm.

3. Decisions

1
Land both fixes in one PR. They share a code site and a test block. Splitting doubles the review surface for no benefit.
2
Fix A is the priority. The accepted-wrong-guess is the worse user experience — players got credit for a wrong answer. Fix B is a quality-of-life improvement but not a correctness bug for the leaderboard.
3
Acronym mechanism stays algorithmic. No per-game "gta" alias dumping ground. Derived from the target's base name at match time.
4
"gta vc" / "gta sa" stay as per-game aliases. They are sub-acronyms of the subtitle, not the base name; not worth a second acronym layer.
5
No threshold changes. The existing JW/token-sort/hard-floor constants are well-tuned. Both fixes work by adding structural rules, not by moving numbers.
