Hint Rework: From Trivia to Letter Reveal

Date 2026-06-10 · Attendees Game Designer · UX/Product Designer · Backend/Architect · Economy/Monetization Analyst · Facilitator Claude

Players (and the product owner) report the current hints are useless for actually naming the game: revealing release year, publisher, developer, or genre tells you trivia about a game you still can't recognize. All four agents agreed the complaint is valid — it's a category error, not a tuning problem. The fix: add a server-authoritative masked-title letter reveal as a new, scarce, more expensive power-up — without deleting the four metadata hints, so the daily-login economy, existing inventory, and historical stats stay intact.

1. Why this meeting

The classic mode's one job is "name the game from the screenshot" — a recall/recognition task. The current power-ups answer a different question. Four metadata hints exist today (hint_year, hint_publisher, hint_developer, hint_genre — first genre tag only). They unlock only after a wrong guess, cost -20% of the round score, or are free if owned as an inventory item (earned from daily-login streak chests — the day-7 legendary chest grants hint_year ×2, hint_publisher ×2, hint_developer ×1, hint_genre ×1). They are a sink for the login economy and a premium lever (premium = free hints in catch-up only, never the ranked daily).

The complaint: knowing "2011 · FromSoftware · Action RPG" only helps the player who is already 90% to the answer. For everyone else it's a trivia card, delivered exactly when they're already stuck. The proposal on the table: reveal the game title's first letter (or progressively more letters) instead.

2. The personas

Four agents ran in parallel. Condensed verdicts below; full synthesis in §3–§5.

Game Designer Balance, fun, difficulty curve

Verdict

Letter reveal is strictly more helpful and more fun. Metadata answers "facts about a game I can't name"; a masked title attacks recall directly — recognition needs a retrieval cue, and the title's own letters are the strongest cue. A masked skeleton is also a satisfying mini word-puzzle vs reading a trivia card.

Mechanic

  • Show the masked skeleton with word count + lengths (_____ ____ for Elden Ring) — huge constraint, no letters given.
  • Then progressive letter buys; escalating cost; never auto-solve (cap so the title is never fully spelled).

Balance & fairness

  • Letter help must cost more than the flat 20%. Cap stacked penalties so a heavily-hinted answer always ranks below a clean fast solve.
  • Even inventory-granted reveals should still apply the score penalty — otherwise login streaks let whales trivialize the leaderboard.
  • Keep the "first wrong guess" gate — it rewards an honest attempt and prevents reflexive hint-spam.

Edge cases

Mask the fuzzy-normalized title so revealed letters always match a winning input. Articles ("The/Le") shown free; sequel digits ("Portal 2") free or own cheap tier; 1-word titles skip the cheap first-letter tier.

Recommendation

Ship masked progressive reveal as a new hint type. Keep year + developer as cheap secondary hints, deprecate genre from new chests (honor existing inventory). Trade-off flagged: higher completion (good retention) compresses score variance — the leaderboard penalty cap is the load-bearing piece.

UX / Product Reveal pattern, copy, a11y, migration

Verdict

Complaint validated — the reveal is mismatched to the cognitive task. Worse, gating it behind a wrong guess means it arrives once the player has already proven they're stuck.

Pattern: masked title wins

  • Render the dimmed skeleton up front (E____  R___), preserving word breaks + length. Each tap fills the next letter left-to-right. Crossword / Wheel-of-Fortune mental model: legible, satisfying, self-limiting.
  • Single first-letter button = too weak; pure letter-by-letter without shape hides how close you are.

Cost & discoverability

  • Keep the existing badge language — green count for owned/free, red for penalty — but make the penalty per-letter and smaller (e.g. ~8%/letter) so one nudge is cheap and a full unmask is expensive. Show a live running penalty inline.
  • UX would drop the "wrong guess first" gate (punitive, hides the feature's value). conflicts with Game Designer

i18n & a11y

  • Game titles are proper nouns — not localized ("Elden Ring" is identical in fr/en), so revealing Latin letters is correct in both locales. Reveal the canonical accepted title (accents included).
  • Screen reader: don't read raw underscores. Use an aria-label ("Titre masqué : E, puis 4 lettres cachées…") and an aria-live="polite" region announcing each revealed letter; mask glyphs aria-hidden.

Migration

Players earned metadata-hint items; deleting them reads as theft. Convert/announce as an upgrade, surface in the existing rewards-announcement channel, one-time toast on first game after migration.

Backend / Architect Server-authoritative reveal, data model, anti-abuse

The core risk

This deliberately breaks a careful existing invariant: game.service.ts returns availableHints only when isCorrect || isCompleted — never on a wrong guess. Image serving already proxies via /api/game/image/… specifically to hide the slug. Letter reveal leaks a prefix of the answer before a solve, so the masked title must be a metered, server-authoritative leak. The full title must never reach the client.

Endpoint

Dedicated POST /api/game/reveal-letter (body { tierSessionId, position }) — not piggybacked on the guess response. Mirrors how activateSecondChance / POST /api/game/second-chance live outside submitGuess. Returns only the masked string for letters paid for: { maskedTitle, lettersRevealed, cost, nextCost }.

Data model

  • Add 'hint_letter' to PowerUpType + GuessRequest.powerUpUsed.
  • New table position_letter_reveals modeled on position_second_chances (migration 20260511): (tier_session_id, position, letters_revealed int, score_penalty_pct, applied_to_guess_id), UNIQUE(tier_session_id, position). Atomic UPDATE … letters_revealed + 1 RETURNING under withTierSessionLock.
  • Reuse guesses.power_up_used = 'hint_letter' and hint_from_inventory. No new guesses columns.

Masking

Canonical field is gameName. Mask is a pure function buildMaskedTitle(gameName, n) — store only the integer count, recompute the string each call ⇒ idempotent & replay-safe. Article-aware (auto-reveal "The/A/Le"), digits/spaces/punctuation shown free, diacritics preserved.

Scoring

Cost is applied progressively at reveal time (record score_penalty_pct on the row), then deducted in submitGuess after the 200 cap, in the same block as hintPenalty, and before the SECOND_CHANCE_FLOOR clamp so a paid floor still wins. Inventory consumed per-letter via inventoryRepository.useItems(userId,'powerup','hint_letter',1).

Riskiest part — ship gate. The lenient fuzzy matcher (startsWith, acronyms, 0.85 length-ratio) could accept a near-empty guess once a few leading letters are revealed. Cap letters_revealed at min(2, ceil(titleLen × 0.3)) and unit-test that the maximum revealed prefix can never by itself satisfy isMatch. That test gates the release.

Compat

Keep the old hints — advanced-stats hard-codes the four hint_* keys; extend the whereIn + accumulator additively with hint_letter. Add the chest grant via the same idempotent mergeItems pattern.

Economy / Monetization Re-tier, don't replace; protect the ranked board

Verdict

Don't replace — re-tier. A first-letter reveal often cracks the game, so it must cost more and be inventory/premium-gated, never a spammable flat-penalty toggle on the ranked daily.

Why a flat-toggle would break things

  • A -20% penalty against a hint that wins the round is a trade everyone takes daily — the sink stops draining and starts minting score.
  • Free/buyable letter reveals on today's daily convert a skill ranking into a willingness-to-spend ranking.
  • If letter reveal is freely toggleable, the metadata hints inside chests become strictly inferior junk — chest value craters.

Guardrails

  • Convex per-letter penalty floored near second-chance level; inventory-gated on the ranked daily (no inventory → 402 upsell, like second-chance).
  • Penalty applies whether or not you own the item — ownership removes the metadata-hint penalty, never the letter penalty.
  • Premium letter help only in catch-up / casual, never today's leaderboard (mirror the existing hint rule). Paywall on volume/convenience, not rank.
  • Transition: grandfather + convert existing items, sweeten with points, reweight the day-7 chest to feature hint_letter as its marquee item.

3. Consensus & the one disagreement

Unanimous across all four agents:

  • The complaint is valid. Metadata hints answer the wrong question; letter reveal attacks the recall task directly.
  • Add, don't replace. Letter reveal is a new scarce top tier. The four metadata hints stay — deleting them breaks the login economy, orphans earned inventory, and invalidates historical stats.
  • Masked-title skeleton (word count + lengths shown), progressive left-to-right reveal, never auto-solving the full title.
  • Server-authoritative via a dedicated reveal endpoint; the client never receives the full title.
  • Costs more than 20%, convex, gated. Inventory-gated on the ranked daily; premium help confined to catch-up/casual; stacked penalties capped so hinted answers rank below clean fast solves.
  • Hard reveal cap unit-tested against the fuzzy matcher is the ship gate.
Unresolved — the "first wrong guess" gate. Game Designer wants to keep it (rewards an honest attempt, blocks reflexive spam). UX wants to drop it (punitive; hides the feature's value; the skeleton should be visible from the start). Recommendation to the PO: show the masked skeleton from the start (UX), but keep the first paid letter gated behind one wrong guess (Game Designer) — a hybrid that surfaces value without removing the honest-attempt gate. Needs your call.

4. Proposed design

Mechanic

  • Masked skeleton shown (word breaks + per-word length). Articles auto-revealed; digits/punctuation shown free.
  • "Révéler une lettre" fills the next masked position left-to-right. Reveal capped at min(2, ceil(titleLen × 0.3)) so the prefix can never satisfy the fuzzy matcher.
  • Mask built from the fuzzy-normalized canonical title so every revealed letter is something a winning guess would contain.

Cost model (to finalize with PO)

Convex per-letter penalty, e.g. letter 1 ≈ -15%…-25%, letter 2 ≈ -35%…-45%, floored near the second-chance level (~70 pts) so it never zeroes out but never beats a clean solve. Owning a hint_letter item waives the currency (inventory) cost but the score penalty still applies. On the ranked daily, letter reveal is inventory-gated (empty → 402 upsell); premium-free letter help is confined to catch-up/casual.
