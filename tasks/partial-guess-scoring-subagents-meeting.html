Partial Guess Scoring

Subagent design review · 4 personas + devil's advocate · 2026-06-14

Problem

A player identified the franchise from a screenshot but omitted the sequel number or full subtitle, and scored 0. From the reported session:

  • metal gear solid → Metal Gear Solid 2: Sons of Liberty — rejected.
  • final fantasy → Final Fantasy XII — rejected.
  • pokemon → Pokémon X, Y — rejected.

Owner ask (FR): validate these at a lower score; award a bonus when the number / full title is also right. Genuinely wrong guesses (final fantasy vii for FF VIII) must stay rejected.

Persona positions

Product / Game design. Three tiers: exact / close / franchise. Franchise-only is a real reward but unmistakably worse. Speed still rewards skill. Crisp fairness line: omission = partial credit, commission (wrong number) = a miss.
Backend architect. Add evaluateMatch() → { matched, precision }; keep isMatch as a thin wrapper so only the scorer consumes the grade. Map precision→score in game.service via a constant. Apply the partial factor after the 200 cap, before letter penalty & second-chance floor. Do not loosen the letter-reveal leak gate.
QA. Keep all ~90 boolean assertions green; add a parallel graded suite. Highest-risk deltas: the witcher/portal/fallout trio (false → partial) and the leak gate. Pin "wrong number stays none" and "revealed letters can't auto-win".
Devil's advocate. Block as originally framed. The matcher rejects franchise-only on purpose in 5 branches. Hard constraints: partial NEVER on DLC base-name, incomplete truncation, or wrong number; never trivialize the leaderboard via franchise-spam; the leak gate must stay strict; one boolean tier, not a numeric gradient.

Decision

Single partial tier. The matcher gains evaluateMatch(input, name, aliases) → { matched, precision: 'exact'|'partial'|'none' }. isMatch keeps its current exact-only meaning — so the letter-reveal leak gate (effectiveMaxReveals) and every other caller are byte-for-byte unchanged. Only submitGuess reads the grade.

When partial fires (narrow, by construction)

All must hold, after the strict matcher has already said "no":

  • target has a sequel number (so there is something to omit);
  • input matches the franchise root (number stripped) ≥ 0.85;
  • every meaningful input token is covered by a target token — no foreign specifier (kills pokemon diamond → Pokémon X, Y);
  • input gives no wrong number;
  • it is not a DLC base-name match.

Scoring curve

exact = round(min(100 × speed, 200)) (unchanged). partial = round(exact × 0.40). The 0.40 factor guarantees the fastest partial (80) is always below the slowest exact (100), so full identification is strictly the best play. Order: speed → 200 cap → partial factor → letter penalty → second-chance floor.

partial range: 40 – 80     exact range: 100 – 200   (no overlap)

Devil's-advocate constraints — disposition

Honored: partial never touches DLC base-name, incomplete truncation, or any wrong number; the leak gate keeps strict isMatch; single boolean tier, no gradient; partial reveals the answer only because it solves the position (a teaching moment), never on a miss.

Token-coverage guard answers the high-cardinality-franchise abuse: a franchise-only partial must be a subset of the target's words, so naming a different entry ("pokemon diamond") stays a miss.

Consciously overruled: the "non-ranked only" guardrail. The owner explicitly wants partial credit to count; the 0.40 factor (and the no-overlap property) is the mitigation. Revisit if farming shows up in the data.
