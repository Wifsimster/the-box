Senior meeting — Legacy hint retirement & letter-reveal dock integration

2026-06-11 · Chair: Claude (session agent) · Seats: Game Economy Designer, Staff Frontend/UX, Staff Backend, Senior QA/A11y/i18n · Supersedes the relevant decisions of hint-letter-reveal-subagents-meeting.html (2026-06-10).

Why this meeting

The Product Owner believed the four legacy metadata hints (year / publisher / developer / genre, −20% each) had been removed when the masked-title letter reveal shipped. They had not — the 2026-06-10 meeting explicitly decided "add, don't replace" to protect the login economy. The PO's new directive overrides that: only the letter reveal survives. Second complaint: the letter-reveal UI (mask + "Révéler une lettre" pill) floats two rows above the guess field, visually disconnected from where the player actually types.

Ground truth established before debate

  • Letter reveal: server-authoritative POST /reveal-letter; penalty steps [15, 20] ⇒ cost {0, −15%, −35%}; hard cap 2 letters/title; consumes a hint_letter item on the ranked daily and still costs score.
  • Legacy hints: flat −20%, but free when paid from inventory — a hidden rich-get-richer asymmetry favouring streak players.
  • Seven grant sources mint legacy keys (the brief assumed four): daily-login calendar (days 1/2/4/5 + day-7 chest), signup starter inventory (auth.ts: hint_year ×2 + hint_publisher ×1 on every registration), referral (both sides + hardcoded FR email copy), geo-consensus pins, geo first-pin reward, reactivation chest.
  • timer_extension is a dead key: granted by geo-consensus, consumed by nothing anywhere in backend or frontend.
  • HintButtons.tsx owns the fetchInventory() on-mount effect that MaskedTitle silently depends on — naïve deletion breaks ranked-daily inventory state.
  • The "hint-free game" achievement is defined as powerUpUsed !== null; once that column is always null, every game becomes "hint-free" unless letter reveals are counted too.

Decisions

D1 — Full retirement. The four metadata hints leave both the play surface and the economy. UI components HintButtons.tsx and the revealed-hint chips in GuessInput.tsx are deleted; the submitGuess penalty/inventory path is removed. Explicitly overrides 2026-06-10 "add, don't replace".
D2 — Convert, never delete. One ledgered, idempotent migration (20260613_retire_legacy_metadata_hints.ts): pooled conversion 2 legacy → 1 hint_letter, rounded up, minimum 1 if the user held any. Atomic DELETE … RETURNING CTE so concurrent old replicas can't double-spend; legacy_hint_retirement_log audit table powers support queries and an exact down(). Announced through the existing rewards-inbox grant flow (idempotent on sourceRef, socket event + inbox card for free) — framed in French as an upgrade, never a removal. No conversion cap: score cost, not item count, is the leaderboard invariant.
D3 — Retire timer_extension in the same pass. Dead key with live grants reads as a player-facing bug. Existing inventory converts 1:1 → streak_freeze; geo-consensus "every 10th pin" grant becomes streak_freeze ×1.
D4 — Grant rewiring (all seven sources).

D5 — Fused guess field (UX Option B). One bordered shell: a slim mask bar fused to the input's top edge, sharing the existing neon frame (border-primary/30 + shadow-[var(--glow-md)], lifted from the <Input> onto a wrapper with focus-within:). The reveal button is a labelled pill at the bar's right end — stacked above send, never beside it (mis-tap risk on ≤360px). onPointerDown preventDefault keeps the mobile keyboard open. Logic moves to a useLetterReveal hook (including the inventory fetch rescued from HintButtons); MaskedTitle.tsx and HintButtons.tsx die; a presentational LetterRevealBar.tsx renders inside GuessInput. Freed dock height goes to the screenshot (flex-1 absorbs it) — nothing new fills it. Rejected: crossword-style typing into the mask (breaks fuzzy matching — players legitimately type "gta 5"), reveal icon beside send (fat-finger factory under a 45s timer).
D6 — Button states are visible, not tooltip-only. Always-rendered text label carrying the cost (Lettre −15% / Lettre ×2); aria-disabled + tap-fires-toast instead of a dead disabled button (touch users currently get zero explanation of the locked gate); cap-reached unmounts the button leaving the penalty tag as residue; per-letter reveals announced in the existing polite live region ("Lettre R révélée…"). Reveal animation uses the sanctioned reward-reveal bounce, zeroed under prefers-reduced-motion. Stable test-ids: letter-reveal-button, masked-title, letter-penalty-badge.
D7 — Anti-leak: stop sending availableHints. buildAvailableHints dies; correctGame already carries everything the result UI needs. SubmitGuessRequest.powerUpUsed, GuessResponse.hintPenalty / hintFromInventory / availableHints and PositionState.hint*Used leave the shared types. /guess accepts and ignores a stale client's powerUpUsed — never 400 a guess over a retired optional field.
D8 — History is sacred. guesses.power_up_used stays untouched; session-details penalty display and the recalculation worker keep understanding legacy values (marked // LEGACY — historical rows only); advanced stats fold the four keys into one legacyMetadataHints rollup beside hint_letter.
D9 — Achievement integrity. "Hint-free game" extends to require zero position_letter_reveals rows for the session, otherwise retirement makes it trivially earnable.
D10 — QA gates before merge. Rewrite the lying test (daily-game.spec.ts "should display hint buttons" keeps passing against any disabled button); replace locator('..') parent-traversal submit lookups with role+name; delete the dead clickHint helper; new letter-reveal coverage (locked gate, happy path, cap, no-inventory ranked daily, server-authoritative race); ~18 dead game.hints.* i18n keys removed and new keys land in fr + en in the same PR (incl. the how-to-play copy still citing the −20% economy); migration conversion gets its own idempotency test.

Dissents on the record

Economy → PO: retirement removes the game's only zero-score-cost help path; that's regressive for the bottom of the skill curve — the cohort the login economy retains. Pre-negotiated mitigation if completion rates dip: first letter of the day becomes score-free (a penaltyStep-0 carve-out), not a return of metadata hints.
UX → PO: a cost-bearing, irreversible action now lives one tap from the typing surface under a 45s timer. Accepted because the cost label is always visible and the gate locks it by default. If telemetry shows reveals spiking within ~300ms of a failed send tap, the fix is a brief post-submit disabled window — not a confirmation dialog.
Backend → team: during the rolling-deploy window, old replicas run the penalty path against already-drained inventory — a user who "owned" a hint eats −20% on the ranked daily. Deploy at the daily-reset low-traffic point; keep the window short.
QA → PO: instrument daily completion rate, correct-guesses/session and reveal usage segmented by skill quartile, with a pre-removal baseline. The reveal pill (~36px) matches the existing send button but sits below the 44px AAA touch floor — accepted as consistent-with-current, revisit both together. Chair also overruled QA's persistent inline helper text in favour of tap-toast + aria-describedby (dock height is precious with the keyboard open).
