Streamer Kit

Wire The Box into your stream — chat commands, live overlays, and webhook-driven shoutouts.

The Streamer Kit is a read-only, opt-in public API. Your viewers can see your score, streak and rank; nobody can play for you. Three integration paths — a chat command, an OBS overlay, and webhooks — all from one API key.

1. Get set up (3 steps)

Turn on your public profile.
Profile → Streamer Kit → enable Public Streamer Profile. Off by default — nothing about you is queryable until you flip this.
Claim a slug.
3–32 characters, lowercase letters / digits / dash / underscore. This is your public handle — e.g. wifsim → /streamers/wifsim.
Generate an API key.
Shown once. Copy it into your bot or overlay. Lost it? Revoke and make a new one.
Base URL: https://the-box.battistella.ovh/api/public/v1. Every example below uses the placeholder YOUR_SLUG and YOUR_KEY — swap in your real values.

2. Chat command

The fastest win. A viewer types !box and the bot replies with your score line. The profile endpoint has a ?format=chat mode that returns a ready-made single line — no JSON parsing required. 

GET /api/public/v1/streamers/YOUR_SLUG?format=chat

🎮 @Wifsim · Today: 7,820 pts (#42) · Streak: 11d
Nightbot — add a custom command

In the Nightbot dashboard → Commands → Custom → Add Command:

!commands add !box $(urlfetch https://the-box.battistella.ovh/api/public/v1/streamers/YOUR_SLUG?format=chat)

Use urlfetch (not urlfetch json) — the chat format already returns plain text. Append &emoji=0 if your chat strips emoji.

Streamer.bot — Fetch URL sub-action

Add a command, then a Fetch URL sub-action:

URL:    https://the-box.battistella.ovh/api/public/v1/streamers/YOUR_SLUG?format=chat
Method: GET
Output: %theBoxLine%

Then a Twitch → Send Message sub-action with body %theBoxLine%.

3. Live overlay (OBS)

A browser-source overlay that ticks up with your score, screenshot-by-screenshot. The live endpoint is a Server-Sent Events stream — works with the browser's built-in EventSource, no library needed. Because EventSource can't send headers, the key goes in the query string here (this is the only endpoint where that's allowed). 

<!-- overlay.html — drop into an OBS Browser Source -->
<div id="score">—</div>
<script>
  const KEY  = 'YOUR_KEY'
  const SLUG = 'YOUR_SLUG'
  const es = new EventSource(
    `https://the-box.battistella.ovh/api/public/v1/streamers/${SLUG}/live?key=${KEY}`
  )
  es.addEventListener('screenshot.scored', (e) => {
    const s = JSON.parse(e.data)
    document.getElementById('score').textContent =
      `${s.score} pts · ${s.screenshotsDone}/10`
  })
  es.addEventListener('session.completed', (e) => {
    const s = JSON.parse(e.data)
    document.getElementById('score').textContent =
      `Final: ${s.score} pts (#${s.rank})`
  })
</script>

The stream emits connected on open, screenshot.scored on every progress change, session.completed when you finish, and a heartbeat every 15s. It auto-closes 30s after completion.

4. Webhooks

For "post my result in chat the moment I finish" — register a webhook URL and The Box POSTs a signed JSON payload to it. Register either from Profile → Streamer Kit, or with your key: 

curl -X POST https://the-box.battistella.ovh/api/public/v1/webhooks \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://hooks.example.com/the-box","label":"Discord bot","events":["session.completed"]}'

The response includes a secret (shown once). Every delivery is signed — verify it before trusting the payload: 

import crypto from 'node:crypto'

function verify(rawBody, header, secret) {
  const p = Object.fromEntries(header.split(',').map(x => x.split('=')))
  const expected = crypto.createHmac('sha256', secret)
    .update(`${p.t}.${rawBody}`).digest('hex')
  return crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(p.v1))
}
// header = request.headers['x-thebox-signature']
Heads up. URLs must be HTTPS and publicly reachable — the SSRF guard rejects private / loopback / cloud-metadata addresses. De-duplicate deliveries on eventId; a retried delivery sends the identical body.

5. Limits & rules

ThingLimit
Anonymous requests60 / min per IP
Keyed requests600 / min per key
Concurrent SSE streams3 per key
Active webhooks10 per account
Webhook retries3 (15s / 60s / 240s)

Every response carries RateLimit-Limit / RateLimit-Remaining / RateLimit-Reset so your bot can self-throttle. A 429 means back off for Retry-After seconds.
